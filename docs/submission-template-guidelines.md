# Submission Template Guidelines

Students submit two files for the MVP:

- One Word `.docx` report.
- One Draw.io `.drawio` architecture diagram.

## Word Report

Use clear section headings. The parser groups paragraphs under headings and sends those sections to AI grading.

Recommended headings:

- Project Overview
- Requirements
- Architecture Design
- Database Design
- API Design
- Error Handling
- Security
- Deployment

Avoid putting important content only in images. Text inside images is not extracted in the MVP.

## Draw.io Diagram

Use labeled shapes and connectors.

Recommended diagram practices:

- Give every important component a readable label.
- Connect related components with arrows/connectors.
- Label connectors when the relationship is not obvious.
- Keep architecture diagrams in the `.drawio` XML format.

## Current MVP Limits

- No OCR for screenshots inside Word files.
- No plagiarism detection.
- No automatic grade publication.
- No batch upload for a whole class yet.
